# 入力書式リファレンス

Web UI（Gradio）のテキスト欄に入力する本文の書き方まとめ。
Markdown 風の記法で、見出し・数式・表などを手書き風に組版する。

実装: `src/layout/typesetter.py`（`typeset()` が第1パスで段落解析 → 各ブロックを配置）。

---

## 見出し

行頭に `#` を付ける。最大3段階で、階層ごとに左インデントが深くなる。

```
# 大見出し
## 中見出し
### 小見出し
```

- フォントは見出しレベルで拡大（1.15 / 1.08 / 1.0 倍）。
- 見出し直後の本文も同じインデント階層に揃う。

## 段落

- **空行**で段落を区切る。
- 段落内は本文幅で自動折り返し（禁則処理あり）。

## 数式

| 種類 | 書き方 | 説明 |
|---|---|---|
| インライン | `$V = IR$` | 文中に埋め込む |
| ブロック | `$$E = mc^2$$` | 独立行に中央配置。`\\\\` で複数段 |
| 分数 | `$\frac{a}{b}$` | 分子/分母を上下に |
| 上付き/下付き | `$x^2$` / `$f_0$` | 指数・添字（1文字はブレース不要、複数は `x^{10}`） |
| ギリシャ文字 | `$\alpha$` `$\beta$` `$\omega$` | 主要文字に対応 |
| 式番号 | `$$ ... \tag{1}$$` | ブロック数式の右端に番号を併置 |

- 数式内の `.` は小数点として保持（本文の `.` は句点 `。` に自動変換される）。

## 表（Markdown パイプ表）

```
| 材料 | 降伏応力 | 引張強さ |
|---|---|---|
| SS400 | 245 | 400 |
| S35C | 305 | 510 |
: 表1 各材料の機械的性質
```

- **2行目の区切り行 `|---|---|` が必須**（これが無いとただの本文扱い）。
- 列幅は各列の最大文字数から自動決定し、本文幅を超える場合は縮小して収める。
- 表は**本文幅の中央**に配置。横罫線は用紙の罫線に揃う。
- **キャプション**: 表の直後に `: タイトル` の行を書くと、表の真下に中央寄せで描画（Pandoc 式）。省略可。
  - 学術慣例では表のタイトルは上が標準だが、本ツールは下配置。

## 句読点・記号

- 本文の半角ピリオド `.` は句点 `。` に自動変換（数式内は除く）。
- 幾何描画対応記号: `、。・（）「」『』` と ASCII 数式記号 `+ - = < > * / % : ; ! ?`
- 数字(0-9)・英字は専用字形で描画（ML 変形を通さないので崩れない）。

## 対応文字

ひらがな・カタカナ・漢字（常用）・英数字・数式記号。
手書きサンプル収集済みの文字はユーザー筆跡、それ以外は KanjiVG ベースで生成。
プレビュー後「文字カバレッジ」で各文字の描画方式を確認できる。

## 手書きらしさ・実機の設定（スライダー）

書式ではないが、設定パネルで調整できる主な軸。詳細は CLAUDE.md 参照。

| スライダー | 既定 | 効果 | 実機メモ |
|---|---|---|---|
| 温度 | 1.0 | 字形そのものの揺らぎ（ML offset） | — |
| 汚さ | 1.0 | 行内の上下動・字間・サイズ・傾き | — |
| 字のばらつき | 0.5 | 同じ字を毎回少し変える | — |
| 筆圧変化 | 0.0 | 画内の濃淡 | 実機は描画中Zが振れ点線化するため0推奨（プレビュー演出用） |
| 入筆 | 0.0 | 始筆を軽く入れる | 実機は始筆かすれ得るため0推奨 |
| 連綿（続け字） | 0.0 | 同字の近い画を薄く続ける | Z一定なので実機OK。近いほど高確率＋乱数 |

実機（単線シャーペン）は描画中に Z を上下させると点線化するため、筆圧変化・入筆は 0 が安全。
連綿は Z 一定なので実機でも使える。
