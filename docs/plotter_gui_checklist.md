# xDraw A4 G-code 送信 GUI 実機チェックリスト

Windows ネイティブ Python (3.12+) 環境で以下を確認する。
WSL では Tkinter が起動しないので必ず Windows で実行すること。

## 起動
- [ ] `python scripts/run_plotter_gui.py` または `python -m src.plotter_gui` で GUI ウィンドウが開く
- [ ] ウィンドウタイトルが「xDraw A4 G-code Sender」

## ポート検出と接続
- [ ] xDraw A4 を USB 接続した状態で「自動検出」ボタンを押すと、ドロップダウンに COM ポートが選択される
- [ ] 接続ボタンで状態ラベルが「接続中」(緑) になる、ログに `Connected: COM<n>` が出る
- [ ] 切断ボタンで「未接続」(灰) に戻り、ログに `Disconnected` が出る
- [ ] xDraw を抜いた状態で接続を試みるとエラーログが残るが、GUI は落ちない

## 動作確認系
- [ ] 接続後、ホーミングボタンで物理的に左上角へ移動する
- [ ] ペン Up ボタンでペンが上がる、ペン Down ボタンで下がる
- [ ] ホーミング/ペン操作後、ログに `[start] home` `[done] home` のような区切りが残る

## 送信フロー
- [ ] 「ファイル選択」で `.gcode` を選ぶとプレビュー領域に 2D 描画が出る
- [ ] 「送信開始」ボタンを押すと送信が始まり、進捗バーが滑らかに進む
- [ ] 進捗ラベルが `<idx> / <total> 行 (XX%)` の形式で更新される
- [ ] 現在送信行が表示される
- [ ] 送信完了で `[done] stream` ログ、ボタンが元の状態に戻る

## 緊急停止
- [ ] 送信中に「緊急停止」ボタンを押すと、5 行以内に物理動作が止まる
- [ ] ログに `emergency stop` が記録される
- [ ] 緊急停止後、再度ホーミング → 送信が可能（GRBL Soft Reset 後の復帰確認）

## 異常系
- [ ] 未接続で送信開始 → 警告ログ、submit_stream 呼び出されず
- [ ] ファイル未選択で送信開始 → 警告ログ
- [ ] 送信中にウィンドウを閉じても worker thread が固まらず Python プロセスが終了する

## 終了処理
- [ ] ウィンドウ X ボタンで閉じても異常終了しない
- [ ] Ctrl+C ターミナル中断でも安全に終了する
